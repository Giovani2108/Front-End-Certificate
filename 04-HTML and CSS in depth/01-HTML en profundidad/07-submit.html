07 - Envío de Formularios: action y method

La etiqueta <form> envía datos al servidor. Para controlar cómo y dónde se envían usamos:

  • action: URL donde se envían los datos.
  • method: método HTTP para enviar los datos, GET o POST.

Tipos de método

GET: envía datos en la URL (visible). Útil para búsquedas o datos no sensibles.

POST: envía datos en el cuerpo de la petición (no visible). Ideal para información privada, como contraseñas.

Ejemplo GET

<form action="/buscar" method="get">
  <input type="text" name="q" placeholder="Buscar..." />
  <input type="submit" value="Buscar" />
</form>

Ejemplo POST

<form action="/login" method="post">
  <input type="text" name="usuario" placeholder="Usuario" required />
  <input type="password" name="clave" placeholder="Contraseña" required />
  <input type="submit" value="Entrar" />
</form>

Si no pones method, el formulario usa GET por defecto.
