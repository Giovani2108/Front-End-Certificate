04 - Mejorando la Semántica HTML

Evita usar contenedores genéricos como <div> para todo. En su lugar, utiliza etiquetas semánticas que describen el propósito del contenido.

¿Por qué usar etiquetas semánticas?

  • Claridad estructural: Describe mejor la intención del contenido.
  • Accesibilidad: Los lectores de pantalla y buscadores interpretan mejor la página.
  • Mantenibilidad: El código es más legible para otros desarrolladores (¡y para ti en el futuro!).
  • SEO: Mejora la forma en que los motores de búsqueda entienden tu contenido.

Ejemplo sin semántica (NO recomendado)

<div class="header">
  <h1>Welcome to My Blog</h1>
</div>

<div class="nav">
  <ul>
    <li><a href="#about">About</a></li>
  </ul>
</div>

<div class="content">
  <div id="about">...</div>
  <div class="article">...</div>
</div>

<div class="footer">
  <p>Contacto...</p>
</div>

Ejemplo con etiquetas semánticas (RECOMENDADO)

<header>
  <h1>Welcome to My Blog</h1>
</header>

<nav>
  <ul>
    <li><a href="#about">About</a></li>
  </ul>
</nav>

<main>
  <section id="about">
    <h2>About Me</h2>
    <p>Breve introducción sobre el autor.</p>
  </section>

  <article>
    <h2>Últimos artículos</h2>
    <article>
      <h3>Artículo 1</h3>
      <p>Resumen del artículo.</p>
    </article>
  </article>
</main>

<footer>
  <p>Contacto: email@example.com</p>
</footer>

CSS más limpio

Con semántica, puedes evitar clases innecesarias y seleccionar directamente por etiqueta:

/* En lugar de usar .header, .nav, .content */
header {
  background-color: #f0f0f0;
}

nav ul {
  list-style: none;
}

footer {
  text-align: center;
}
